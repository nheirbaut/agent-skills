# Solid Skills

An OpenCode skill for professional software engineering — SOLID principles, TDD, clean code, design patterns, and architectural best practices.

## What it covers

- **SOLID principles** — SRP, OCP, LSP, ISP, DIP with practical decision questions
- **Test-Driven Development** — Red-Green-Refactor, the three laws of TDD
- **Clean code** — naming, structure, value objects, object calisthenics
- **Design patterns** — creational, structural, and behavioral patterns
- **Architecture** — vertical slicing, horizontal decoupling, the dependency rule
- **Code smells** — detection and refactoring strategies
- **Complexity management** — YAGNI, KISS, DRY (Rule of Three)
